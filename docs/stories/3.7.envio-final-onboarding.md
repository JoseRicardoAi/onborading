# Story 3.7: Envio Final Do Onboarding

## Status

Review

## Story

**As a** funcionario,
**I want** revisar e enviar meu onboarding por completo,
**so that** o RH receba meu cadastro finalizado e o link nao aceite mais alteracoes.

## Acceptance Criteria

1. Formulario exibe uma etapa de revisao antes do envio final.
2. O envio final exige confirmacao explicita do funcionario.
3. O sistema valida os dados obrigatorios antes de concluir.
4. Ao concluir, o cadastro muda para `cadastro_completo`.
5. Ao concluir, o token fica marcado como utilizado e nao aceita nova edicao.
6. O funcionario recebe uma tela de sucesso apos o envio final.
