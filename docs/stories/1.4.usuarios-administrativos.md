# Story 1.4: Usuarios Administrativos Do RH

## Status

Review

## Story

**As a** operador do RH,
**I want** ter meu proprio usuario e senha no sistema,
**so that** eu consiga cadastrar e acessar dados de funcionarios sem compartilhar uma unica credencial administrativa.

## Acceptance Criteria

1. Usuario autenticado consegue cadastrar novos operadores do RH.
2. Cada operador cadastrado possui nome, e-mail, senha e status ativo.
3. Login administrativo aceita usuarios cadastrados no banco.
4. Senhas de operadores nao sao armazenadas em texto puro.
5. A credencial por variavel de ambiente continua disponivel como bootstrap.

## Completion Notes List

- O sistema agora permite cadastrar operadores do RH em `/configuracoes/usuarios`.
- O login administrativo tenta autenticar primeiro pelo banco e, se nao houver usuario valido, usa a credencial de bootstrap por variavel de ambiente.
- As senhas sao armazenadas com hash `scrypt` e salt aleatorio.
- Para producao, apos configurar `DATABASE_URL`, aplique o schema com `npm run db:push:web`.
- A imagem de producao agora tenta executar `Prisma db push` automaticamente antes de subir o servidor, usando `DATABASE_URL`.
