## Story

**As a** RH ou gestor autorizado,
**I want** acessar o painel com credenciais administrativas,
**so that** somente usuarios internos possam visualizar dados de onboarding.

## Acceptance Criteria

1. Usuario administrativo consegue fazer login.
2. Usuario nao autenticado nao acessa painel administrativo.
3. Sessao administrativa pode ser encerrada.
4. Fluxo de erro de login e claro.

## Dev Notes

Relevant source artifacts:

- `docs/prd.md`: Epic 1 / Story 1.2 defines administrative authentication.
- `docs/fullstack-architecture.md`: recommends a simple fullstack app deployable through Coolify.
- `docs/stories/1.1.base-da-aplicacao-web.md`: established the Next.js app, health check and Coolify deployment baseline.

Implementation guidance:

- Use a minimal MVP admin authentication based on environment-provided credentials.
- Use an HTTP-only cookie for the admin session.
- Do not introduce persistent admin users yet; database-backed users belong to later data/auth hardening work after Story 1.3.
- Required runtime variables:
  - `ADMIN_EMAIL`
  - `ADMIN_PASSWORD`
  - `AUTH_SECRET`

### Testing

- Validate `/login` accepts valid credentials and rejects invalid credentials.
- Validate `/admin` redirects unauthenticated users to `/login`.
- Validate logout clears the session.
- Run `npm run lint`, `npm run typecheck`, and `npm run build:web`.
