## Story

**As a** usuario administrativo,
**I want** acessar uma aplicacao web inicial,
**so that** eu saiba que o sistema esta online e pronto para evolucao.

## Acceptance Criteria

1. A aplicacao possui pagina inicial administrativa ou tela de login.
2. A aplicacao roda localmente em ambiente de desenvolvimento.
3. A aplicacao possui configuracao basica para deploy em Coolify.
4. A aplicacao possui health check simples.

## Dev Notes

Relevant source artifacts:

- `docs/prd.md`: Epic 1 / Story 1.1 defines the base app, local dev, Coolify baseline and health check.
- `docs/front-end-spec.md`: Admin login/dashboard direction should be clear, utilitarian and responsive.
- `docs/fullstack-architecture.md`: recommends Next.js, TypeScript, Tailwind, PostgreSQL, Prisma and Docker/Coolify deployment.

Implementation guidance:

- This story should not implement full authentication yet. A login-style landing screen is enough for AC1.
- Use `apps/web` as the first implementation app.
- Keep database-dependent behavior out of Story 1.1.
- The health check must not expose secrets or environment values.

### Testing

- Test file location: `apps/web/tests` or co-located tests in later stories.
- For this foundation story, validation should prioritize typecheck/build and a simple health endpoint check once dependencies are installed.
- Future stories will add Playwright E2E flows.
