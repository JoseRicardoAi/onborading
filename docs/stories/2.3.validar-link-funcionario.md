# Story 2.3: Validar Link Do Funcionario

## Status

Review

## Story

**As a** funcionario,
**I want** abrir meu formulario por link,
**so that** eu possa preencher meus dados com seguranca.

## Acceptance Criteria

1. Link valido abre o formulario correto.
2. Link expirado exibe tela de expiracao.
3. Link finalizado nao permite nova edicao.
4. Link invalido nao revela dados internos.

## Tasks / Subtasks

- [x] Validar token publico contra hash persistido
- [x] Bloquear links expirados, revogados ou finalizados
- [x] Ajustar `/onboarding/[token]` para mostrar estados dedicados
- [x] Persistir dados de uniforme apenas para token valido
- [x] Rodar `npm run lint:web`
- [x] Rodar `npm run typecheck:web`
- [x] Rodar `npm run build:web`

## Completion Notes List

- A tela publica nao revela dados internos quando o token e invalido.
- Tokens finalizados respeitam `used_at` e tambem status `cadastro_completo` ou `revisado`.
