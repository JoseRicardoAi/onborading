# Story 2.2: Gerar Link Unico

## Status

Review

## Story

**As a** RH,
**I want** gerar um link unico para o funcionario,
**so that** ele possa preencher o formulario sem login.

## Acceptance Criteria

1. Sistema gera token unico e nao previsivel.
2. Link fica associado a um cadastro especifico.
3. Link possui data de expiracao configuravel.
4. RH consegue copiar o link.

## Tasks / Subtasks

- [x] Criar token service com hash e expiracao configuravel
- [x] Implementar `POST /api/admin/employees/:id/tokens`
- [x] Exibir geracao de link na lista administrativa
- [x] Exibir campo copiavel para o RH compartilhar o link
- [x] Rodar `npm run lint:web`
- [x] Rodar `npm run typecheck:web`
- [x] Rodar `npm run build:web`

## Completion Notes List

- Token raw nunca e persistido; apenas hash com secret.
- Links anteriores ativos do mesmo funcionario sao revogados ao gerar novo convite.
- O TTL padrao usa `ONBOARDING_TOKEN_TTL_DAYS` com fallback de 7 dias.
